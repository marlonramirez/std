Finalmente sfx ha dejado de ser Beta para hacer parte formal de la libreria, con el ahora objeto se pueden realizar animaciones como transiciones, efecto drag and drop o fade; su utilización se ha simplificado, para dejar de lado al operador new y la creación de objetos mediante constructor. Actualmente cualquier transición se efecuta con el metodo anim pasando tres parametos, uno de ellos opcional; dichos parametros son: element, props y opt. Element es el emento al cual se le va a aplicar la animación, props es un objeto con las propiedades que se desea animar y los valores finales de la animación, por ultimo opt el objeto con los parametros opcionales de la animación; estos parametros son: duration, fps y onComplete. Duration es el tiempo que va a durar la animación, fps los frames por segundo que interpolara la animación y onComplete es una función que se efecutara al termino de la animación. Acontinuación se vera un ejemplo de animación con sfx:

Funcionamiento actual de sfx
evt.add($("#target"),{
	click: function () {
		if(css(this).get("width") == "20px") {
			sfx.anim(this, {
				width:"100px",
				height:"100px"
			}, {
				duration:500
			});
		} else {
			sfx.anim(this, {
				width:"20px",
				height:"20px"
			}, {
				duration:500
			});
		}
	},
	mouseover: function() {
		sfx.anim(this, {
			backgroundColor: "#F00"
		}, {
			duration: 500
		});
	},
	mouseout: function() {
		sfx.anim(this, {
			backgroundColor: "#0CC"
		}, {
			duration:500
		});
	}
});

Se han superado algunas limitaciones gracias al nuevo engine css std, los problemas superados son los siguientes:

  • Fade de colores (es decir cambiar gradualmente de un color a otro)
  • cambio de animación sobre la marcha (es decir obtener los datos css del elemento y segun esta información realizar la transición hacia las nuevas propiedades)

Estas limitaciones se lograron superar sin abandonar en ningun momento los avances que se habian logrado obtener con la versión beta, ademas de mantener el pequeño peso de la libreria (Lo cual siempre a sido una de las prioridades de std.js). Se evaluo en algun momento del desarrollo el abandono de sfx y la inclusión de una libreria de terceros como emile.js la cual maneja las funciones antes mencionadas pero abandonaba algunas funcionalidades que posee sfx como lo son el manejo de tranparencias, por lo tanto tras varios estudios de rendimiento, compresión y gracias al nuevo motor css se mejoro el objeto sfx. evt (Antiguamente Event) tambien tiene novedades, aparate de la reestructuración interna a un nuevo patron de diseño, cuenta con la inclusión de un nuevo metodo on que sirve para delegar a un elemento padre las acciones que se puedan realizar sobre sus elementos hijos, tiene como principal pilar el burbujeo del evento, por lo cual no son soportados los eventos que no burbujean. La principal función que desempeña este nuevo metodo dentro de std.js, es la de colocar a escuchar al documento todos los elementos anchor que posean atributo rel="modal" o rel="external", de esta manera cualquier nuevo elemento creado dinamicamente que posea estos atributos, puede accedes a las funcionalidades javascript que estos conllevan.

A raiz de algunos proyectos me he dado cuenta que el metodo on del objeto evt no esta funcionando como deberia, por lo tanto este espacio esta destinado para realizar las pruebas necesarias para que este metodo funcione correctamente.

Se ha solucionado el problema presentado con el nuevo metodo de evt pero su estructura a cambiado en favor de solucionar dicho inconveniente, siempre es bueno poseer un betatester para este tipo de tareas (deteccion de comportamientos no deseados o bugs), pero en ocaciones no es posible y los errores se hallan dentro del entorno mismo de producción, como ha sucedido en esta ocación. El problema radicaba en que al pulsar sobre un nodo, el elemento delegado para observar el comportamiento sus hijos solo observaba el comportamiento del nodo seleccionado y si este nodo no tenia relación con los elementos a observar no se realizaba ninguna acción, hasta ahi se puede decir que el comportamiento es el deseado, pero no es así, supongamos el siguiente caso:

Comportamiento no deseado en on
<div id="observador"">
	<a rel="modal"><img src="images/widget.png" /></a>
</div>
MirdWare © 2012

En el caso planteado al pulsar sobre la imagen no se producira ningun tipo de efecto, dado que no se ha clickeado sobre el enlace si no sobre la imagen. Para solucionar esto se debe implementar un burbujeo hacia los padres del elemento pulsado, esto con el fin de revisar si algun padre del elemeto pulsado esta siendo observado. En este caso, se implementa un ciclo para ir revisando los padres del elemento de la siguiente manera:

Metodo on del objeto evt
on: function(element, observe, nEvent, fn, capture) {
	if(loops(element, nEvent, fn, capture, observe)) {
		core.add(element, nEvent, function() {
			var t = core.get().target;
			if(observe == "*") {
				fn.apply(t, arguments);
			} else {
				while(t && t !== element) {
					if(t.nodeName == observe.toUpperCase() || "#"+t.id == observe || 
					"."+t.className == observe || "@"+t.name == observe) {
						fn.apply(t, arguments);
					}
					t = t.parentNode;
				}
			}
		}, capture);
	}
}

Como hemos observado las ventanas speudomodales cuenta con un nuevo metodo de apertura que hace uso del ya mensionado evt.on(), con lo cual logra abrir ventanas modales de elementos creados dinamicamente, para esto se ha reestructurado el sistema de apertura en ready().

Antiguo sistema de apertura modal
var anchors = $("a");
for(var i=0,a; a=anchors[i]; i++) {
	if (a.href) {
		if (a.rel) {
			if (a.rel == "external") {
				a.target = "_blank";
			} else if (a.rel == "modal") {
				evt.add(a,"click",showModal);
			}
		}
	}
}
MirdWare © 2012

Ahora cuando se carga el DOM, mediante ready() se delega al elemento document los eventos sucedidos en las etiquetas anchors, de esta manera no importa cuando, ni donde se creen las etiquetas dentro del documento.

Nuevo sistema de apertura modal
evt.on(document,"a","click", function() {
	var rel = this.rel;
	if(rel == "modal") {
		modal.show.apply(this,arguments);
	} else if(rel == "external" && this.target != "_blank") {
		this.target = "_blank";
	}
});
